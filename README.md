# YouTube — OSRS Video Essay Workshop

Working repo for researching, developing, and writing game-design video essays about Old School RuneScape, in the voice the channel is moving toward (see `exurb1a.md`).

## How the pieces fit

| File / folder | What it's for |
|---|---|
| `CLAUDE.md` | The operating instructions. Channel identity, the anti-AI writing rules, craft, workflow, accuracy guardrails. Auto-loads every session. **The hard floor.** |
| `exurb1a.md` | The target voice — tonal formula, prose mechanics, structure to reach for on top of the rules. |
| `ideas/backlog.md` | The idea bank. Every spark lands here as a thesis before it becomes a script. Nothing gets lost. |
| `research/osrs-facts.md` | Verified-facts ledger. Numbers, dates, formulas with a source and a date-checked. Consult before scripting; never script a stat that isn't in here or freshly verified. |
| `reference/script-template.md` | The essay spine as a fill-in template with `[VISUAL: ...]` cues. Start every draft from this. |
| `reference/channel-context.md` | What the channel has already published, its recurring positions, and how hard to push the exurb1a voice per video. Keeps scripts consistent and cross-referable. |
| `scripts/drafts/` | Work-in-progress scripts. |
| `scripts/final/` | Approved, ready-to-record scripts. |

## The loop

1. Spark → `ideas/backlog.md` (working title + one arguable thesis).
2. Pick one → clarify the angle (propose 2–3 theses if broad).
3. Verify mechanics → `research/osrs-facts.md`. Flag anything uncertain.
4. Outline on the spine → `reference/script-template.md`.
5. Draft in `scripts/drafts/`.
6. Self-audit against `CLAUDE.md` §2, then voice-pass against `exurb1a.md`.
7. Approved → `scripts/final/`.

When two docs pull against each other, `CLAUDE.md`'s anti-AI and accuracy rules win; `exurb1a.md` is the voice we layer on top.
