# Project M — Ten MMO Bridge Essays

Ten complete scripts commissioned after the GPT dialogue essay. The batch keeps Old School RuneScape as the evidence base while widening the channel toward MMO design, interface history, online communities, monetization, and memory.

Every script carries the full house pipeline:

**locked thesis → engagement map → complete narration → sparse delivery/visual notation → music direction → five ranked SEO packages → fact flags → pre-delivery audit**

The scripts follow `CLAUDE.md`, `exurb1a.md`, `reference/script-template.md`, `channel-context.md`, and the pivot rules already established in the repository: standalone provocation, stranger-legible packaging, mechanics before philosophy, a serious counterargument, calm visual pacing, and no “part N” framing.

## Why these ten do not duplicate the existing inventory

The repository already has scripts on inventory constraints, Moons of Peril, ticks, attention, polls, GE monetary policy, quests, the XP curve, dead minigames, the wiki, collection logs, death, Ironman, dailies, clues, classlessness, pets, banks as autobiography, Slayer, dry streaks, bots, Leagues, Tears of Guthix, Stronghold of Security, rock cakes/NMZ, sand-crab etiquette, and the Falador Party Room.

This batch deliberately enters different territory:

- `#5` concerns the **public space outside the bank**, while Project V2's bank essay concerns stored objects as autobiography.
- `#9` concerns **privatized infrastructure and public traffic**, while Project W's boat essay concerns travel and dependence.
- `#3` concerns **equipment ecology**, while the published Moons videos concern encounter-specific damage and defence systems.
- `#7` uses RuneScape as evidence for a **general level-scaling argument**, rather than revisiting OSRS-versus-WoW as a comparison.
- `#10` concerns **relationship persistence**, distinct from bots, clans, or generic nostalgia.

The owner-rejected party-hat and soundtrack pitches remain untouched.

## Verification policy

These drafts are recording-ready in argument, structure, prose, visuals, music, and packaging. They are **not card-ready until each file's fact flags are cleared**.

Priority checks:

1. Current Tutorial Island sequence and arrival inventory.
2. Current free-to-play/member boundaries and the selected gate's text.
3. Every named equipment interaction in the best-in-slot script.
4. Current world-hop rules and which state is local to a world.
5. Ethical capture/anonymization for public chat and friends-list footage.
6. Firemaking's documented historical and current uses.
7. Named level-scaling systems in any non-RuneScape footage.
8. Current context-menu, mobile, and plugin behavior.
9. Every POH feature shown, plus evidence for displaced public routes.
10. Consent for any real relationship story.

No draft invents a statistic to carry its thesis. Unstable mechanics remain numberless or explicitly flagged.
